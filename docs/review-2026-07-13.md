# session-pet Review — Final Report

## 1. Ship-blockers (multi-role consensus, verified)
1. **State-loss races on `.state/state.json`** (Engineer P1 + Tester, all endorsed). `saveState` (SessionPet.swift:85-88) is non-atomic; a torn read by pet.py's statusline (`load_state` → `{}`, pet.py:56-72) atomically persists an empty dict — permanently wiping species/hatched/XP. Fix: write with `.atomic`, `createDirectory` for `.state/` (never created — fresh clones silently lose every species pick/XP; install.sh too), reload before mutate, refuse-overwrite-on-parse-failure with `.bak`, single-instance flock.
2. **events.jsonl replayed on every launch as fresh notifications** (Tester, verified by two roles). `evOffset` starts 0 (line 888); history is re-stamped with current time (line 988) and flips busy sessions to false "input" (line 1000). Fix: seek to EOF at launch, advance offset only to last `\n`, rotate file past ~64KB.
3. **Blocked sessions silently vanish after 5 min** (Product P0 + Engineer P2 converged). `else { continue }` in scanSessions (line 366) drops permission-prompt sessions; the notif merge can't resurrect them. Fix: keep them listed as a new **"stalled"** phase + parse Codex exec/patch approval events in tailInfoCodex. Bonus: retention preserves `prevPhases`, fixing Tester's missed-Glass/XP-after-drop bug for free.
4. **64KB tail misreads** (Engineer + Tester; measured 41→70 oversize lines in one day). Strict UTF-8 decode (tailLines:129-137) nils the whole tail on a mid-char split; a >64KB tool_result hides the decisive event — false "working", no Glass/XP, sticky via mtime-keyed stopCache. Fix: lossy decode + backwards growing-chunk read (cap ~1MB), drop the leading partial line.
5. **`totalXP` ignores `banked_xp`** (SessionPet.swift:90-93). Cross-review found it's guaranteed self-harm: native XP banks under the literal key `"window"`, which pet.py's 300-entry prune will pop into `banked_xp` — the pet demotes (dragon→adult) with no cause. One-line fix.

## 2. Quick wins (ranked; items 1-7 = one-afternoon "correctness batch")
1. Timer into `.common` run-loop mode (line 935) — sprite/polls freeze during panel scroll today.
2. Split Ping from the shared 8s soundDebounce (line ~1041) — needs-input must never be masked by a turn-end Glass.
3. Panel sort by phase priority (input > ready > working > idle, then age) — blocked sessions currently sort *last* and can fall past the 20-card cap (lines 398, 845).
4. newTurn meta filter: skip `isMeta`, `<local-command…>`, `[Request interrupted` — kills phantom "processing your prompt" + fake Glass/XP (lines 161-171).
5. Persistent `!` badge while any unacked `ready` session exists (Designer, one predicate in draw) — plus **per-card ack** instead of togglePanel's ack-everything-on-open (lines 955-957), and key acks so housekeeping mtime bumps don't re-nag (line 1005).
6. plist: add `StandardErrorPath` + KeepAlive-on-crash; replace `fatalError` on missing assets.json with a visible alert — the pet currently dies silently until next login.
7. Drive-bys while in the file: POSIX locale + Gregorian calendar on the Codex DateFormatter and scan extra day-dirs (the multi-day-session miss is the part that bites Lori), clamp the scale arg, decode Codex `exec_command_begin.command` as `[String]`, treat `function_call_output`/`user_message` as decisive.
8. Right-click → NSMenu (Open panel / Sound / Species / Quit) instead of instant `NSApp.terminate` (line 469) — undiscoverable, accidental, unrecoverable with KeepAlive=false.
9. Observe `didChangeScreenParametersNotification` + clamp pet into visibleFrame; position panel against `window.screen`, not `NSScreen.main`.
10. **Test floor** (Tester): `--scan-once` headless flag + fixture transcripts (end_turn, >70KB line, UTF-8 split, meta events, real Codex rollout) + `SESSION_PET_LOG=1` transition log. Gate every fix above on a failing-then-passing fixture.

## 3. Design upgrades (code-island-inspired, in dependency order)
- **Prerequisite first**: in-place card updates keyed by path (Engineer's churn fix — sig currently embeds `fmtAge`/`doing`, lines 848-850, rebuilding all cards at 1Hz). Three roles converged: the live-timer pill, hover tracking, chevron rotation, and 0.18s expand animation are all unsafe until this lands; include in-place `stringValue` updates so age labels don't freeze.
- **No dependency, ship now**: state-tinted card surface — fill `color@0.07/0.04`, 1px border `@0.35/0.15` (0.45 for input), radius 10 (~6 lines in buildCard); neutral header (dot = only colored element, provider mini-badge, right-aligned muted age); designed empty state + first-run "pick who hatches" onboarding while `hatched==false`; colored count chips replacing statusLabel; colorblind dual coding in the dot row (working=circle, ready=square, input=blink); XP bar 220→288pt as a drive-by.
- **After the churn fix**: status pill with live elapsed counter, hover/pointing-hand, chevron + animated expand, snippet in a labeled inset well.
- **Stalled phase visual spec** (Designer): desaturated amber, steady, hollow-ring dot, `?` badge on the sprite — must not collide with "ready".
- **Speech bubble attribution** (Product #3 + Designer spec): pixel-art bubble, one at a time, 5s, stepped shrink, click opens panel pre-expanded; needs window-frame expansion or a child window to avoid clipping.

- **Later — jump-to-session**: strongly endorsed, but after correctness (a jump off a misclassified session burns trust); v1 = activate terminal app + best-effort iTerm2 match, behind an explicit "open ↗" affordance (never state-dependent plain click — Designer); needs the .app bundle for stable TCC grants.
- **Later — .app bundle ticket**: hard prerequisite for TCC stability and any notifications; `pkill -x SessionPet` survives it.
- **Later — hygiene**: background-queue scanning, title/cwd + Codex label caches, stopCache/notif pruning.
- **Rejected — `SessionPet 0` "crash"**: refuted by experiment — AppKit never calls draw for a zero-area view; clamp ships as input polish, not a crash fix (Tester).
- **Rejected — backport native logic into pet_window.py**: permanent 2x tax on an 831-line duplicate; instead move PIXELS out (make assets.json canonical) and mark it deprecated in README (Product + Designer).
- **Rejected — install.sh auto-editing ~/.claude/settings.json**: riskiest write in the project and against the owner's standing rule; print the hook fragment + diff, apply only behind `--write-settings` with backup; fix the `mkdir -p .state` and jq-dependency prerequisites regardless (Engineer + Tester).
- **Rejected — lowering no-signal busyGrace to ~30s**: false "idle" during long tool runs is betrayal; use the no-decisive-event-and-no-prompt→idle rule only (Designer + Product).
- **Rejected/deferred — Notification Center alerts**: off-identity (the pet *is* the notification surface) and `UNUserNotificationCenter` hard-crashes in an unbundled binary; revisit only if the on-screen loop proves insufficient (Designer + Engineer).
- **Rejected — scope traps** (Product, endorsed): 17 providers, in-pet approve/deny, chat-with-agent, notch/menu-bar rewrite, pet creator, more gamification. Add a README non-goals section, plus the false-positive budget: any false "needs input"/"ready" is release-blocking; false "working" is tolerable but time-bounded.
